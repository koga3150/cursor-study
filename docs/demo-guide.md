# Cursor 3.0 デモガイド

## 前提条件

- Cursor 3.x がインストール済み
- Node.js 18+ がインストール済み
- Python 3.10+ がインストール済み（Agent Tab デモ用）

---

## Cursor 3.0 の画面構成

Cursor 3.0 には2つのメイン画面がある:

| | エディタ（IDE） | Agents Window |
|---|---|---|
| 開き方 | 従来のCursor画面 | `Ctrl+Shift+P` → `Agents Window` |
| Agent | サイドパネルの Agent Tab | 複数タブで並列実行 |
| 特徴 | コードを見ながら対話 | エージェント中心の画面 |
| Design Mode | なし | あり |
| Worktree / best-of-n | なし | あり |

デモではこの2つの画面を切り替えながら進める。

---

## 1. Agent Tab デモ（エディタのサイドパネル）

### 概要

Agent Tab はエディタのサイドパネルで動く AI エージェント（旧 Composer の後継）。
コードを見ながら対話的にファイル作成・編集・ターミナル操作を自律的に行う。

### 開き方

- サイドバーの Agent アイコンをクリック
- `Ctrl+I`（Mac: `Cmd+I`）

### デモ手順

#### Step 1: ゼロからアプリを生成

Agent Tab を開き、以下のプロンプトを入力:

```
Python (FastAPI) で簡単な Todo API を作ってください。
- GET /todos で一覧取得
- POST /todos で追加
- DELETE /todos/{id} で削除
- インメモリでデータ保持
- requirements.txt も作成
```

**見どころ:**
- 複数ファイルを一度に生成する様子
- `requirements.txt` の自動生成
- Agent がターミナルで `pip install` 等を自律実行する

#### Step 2: コンテキストを渡して追加機能

```
@main.py に PUT /todos/{id} で更新する機能を追加して
```

**見どころ:**
- `@ファイル名` でコンテキストを明示的に渡せる
- 既存コードの構造を理解して適切な位置に追加
- エラーが出たら自動で修正を試みる

### 実務での活用例

- 新規プロジェクトの雛形生成
- エラーメッセージを貼り付けてデバッグ依頼
- コードを見ながら対話的にリファクタ

## 2. Agents Window デモ（並列エージェント）

### 概要

Agents Window は Cursor 3.0 の新しいメイン画面。
**複数のエージェントをタブで同時に開いて並列実行** できる。
エディタの Agent Tab とは独立した画面で、エージェント中心の操作に特化している。

### 開き方

- `Ctrl+Shift+P`（Mac: `Cmd+Shift+P`）→ `Agents Window`

### デモ手順

#### Step 1: Agents Window を起動

コマンドパレットから `Agents Window` を選択。
**エディタとは別の画面** が開くことを見せる。

#### Step 2: 1つ目のタブでタスクを実行

Agents Window のチャットに以下を入力:

```
Python (FastAPI) で簡単な Todo API を作ってください。
- GET /todos で一覧取得
- POST /todos で追加
- DELETE /todos/{id} で削除
- インメモリでデータ保持
- requirements.txt も作成
```

**Agent が動き始めたら、完了を待たずに次のステップへ進む。**

#### Step 3: 2つ目のタブを開いて並列作業

1つ目の Agent がまだ処理中に、**新しいタブ** を開く。

2つ目のタブで以下を入力:

```
@frontend/src/components/TaskList.tsx のタスクデータに
「コードレビュー」「セキュリティ監査」の2つを追加して。
ステータスはそれぞれ in-progress と todo にして。
```

**見どころ:**
- 1つ目のタブがまだ動いている最中に、2つ目のタブで別作業を開始できる
- 各タブは独立したコンテキストを持つ
- サイドバイサイドやグリッドレイアウトで進捗を同時に確認できる

#### Step 4: 3つ目のタブでさらに並列

さらに新しいタブを開く:

```
このリポジトリの全体像を説明する README.md を書いて。
frontend/ のダッシュボードアプリについても説明を含めて。
```

**見どころ:**
- 3つのタスクが同時並行で進む
- コード生成・コード修正・ドキュメント作成を並列で実行

#### Step 5: 結果の確認

全タブの処理が終わったら、各タブの結果を確認する。

**見どころ:**
- 各タブに会話履歴と変更差分が残っている
- Accept / Reject で変更を個別に取捨選択できる
- エディタ（IDE）に戻ると変更が反映されている

### 実務での活用例

- **マルチタスク:** テスト生成 + リファクタ + ドキュメント更新を同時に依頼
- **試行錯誤:** 異なるアプローチを別タブで試して比較
- **レビュー対応:** 指摘事項ごとにタブを分けて並列修正
- **大規模リファクタ** を裏で回しながら、エディタでは別作業を継続

## 3. Design Mode デモ

### 概要

Design Mode は **Agents Window 内** で使えるモード。
Web フロントエンドのUIをビジュアルプレビューしながら、画面上の要素を
クリック/選択して AI に変更を指示できる。

### 事前準備

```bash
cd frontend
npm install
npm run dev
```

### 開き方

1. `Ctrl+Shift+P` → `Agents Window` で Agents Window を開く
2. `Ctrl+Shift+D`（Mac: `Cmd+Shift+D`）で Design Mode に切り替え

### キーボードショートカット

| 操作 | ショートカット |
|------|--------------|
| Design Mode 切替 | `Ctrl+Shift+D` / `Cmd+Shift+D` |
| エリア選択 | `Shift` + ドラッグ |
| 要素をチャットに追加 | `Ctrl+L` / `Cmd+L` |
| 要素を入力欄に追加 | `Alt+クリック` / `Option+クリック` |

### デモ手順

#### Step 1: Design Mode に切り替え

1. `frontend/` が `npm run dev` で起動していることを確認
2. Agents Window を開く
3. `Ctrl+Shift+D` で **Design Mode** に切り替え
4. ブラウザプレビューが表示される

#### Step 2: ビジュアルで要素を選択して変更

プレビュー画面上のヘッダーをクリックして:

```
このヘッダーの背景色をダークブルー (#1e3a5f) に、テキストを白に変えて
```

**見どころ:**
- 画面要素をクリックするだけで対象コンポーネントを特定
- 変更がリアルタイムでプレビューに反映
- 裏側のコードも自動更新

#### Step 3: レイアウト変更

StatsCard コンポーネントの領域をクリックして:

```
このカードセクションを 3列から 4列のグリッドに変えて、
各カードに軽い影（box-shadow）を追加して
```

**見どころ:**
- レイアウト構造の変更もビジュアル指示で可能
- CSS の知識がなくても自然言語で指定できる

#### Step 4: コンポーネントの追加

画面の空いているエリアを `Shift+ドラッグ` で選択して:

```
ここに円グラフのコンポーネントを追加して。
タスクの完了率を表示するもの。データはダミーでOK。
```

**見どころ:**
- 新しいコンポーネントの追加もビジュアル的に指示
- 配置位置を画面上で指定できる

### 実務での活用例

- デザイナーのフィードバックを画面を見ながらその場で反映
- プロトタイプの素早い作成・修正
- CSSの微調整（余白、色、フォントサイズ等）を直感的に
- 非エンジニアとの画面共有中にリアルタイム修正

## 4. Worktree デモ（変更の分離と best-of-n）

### 概要

`/worktree` は Agent の変更を **Git ワークツリー（別のチェックアウト）で分離** するコマンド。
メインのコードに影響を与えずに実験的な変更を行い、良ければ取り込む。
`/best-of-n` は同じタスクを **複数モデルで同時に実行して結果を比較** する発展形。

どちらも Agents Window 内で使う。

### 前提条件

- Git リポジトリに接続済みであること（このリポジトリは GitHub 連携済み）

### コマンド一覧

| コマンド | 説明 |
|----------|------|
| `/worktree <タスク>` | 別のワークツリーでタスクを実行 |
| `/apply-worktree` | ワークツリーの変更をメインに取り込む |
| `/delete-worktree` | ワークツリーを削除 |
| `/best-of-n <モデル名,...> <タスク>` | 複数モデルで同じタスクを並列実行 |

### デモ手順

#### Step 1: /worktree で分離した環境で作業

Agents Window を開き、以下を入力:

```
/worktree @frontend/src/components/Header.tsx のナビゲーションにドロップダウンメニューを追加して。Projects のリンクにサブメニュー（Active, Archived, All）を付けて。
```

**見どころ:**
- Agent が自動でワークツリー（別チェックアウト）を作成する
- メインのコードは一切変更されない
- 実験的な変更を安全に試せる

#### Step 2: メインのコードが無傷であることを確認

ワークツリーで Agent が作業中（または完了後）に、エディタに戻って `git status` を確認。

```bash
git status
# → nothing to commit, working tree clean
```

**見どころ:**
- メインのチェックアウトには変更が入っていない
- 安心してリスクのある変更を Agent に任せられる

#### Step 3: 変更を取り込む or 捨てる

ワークツリーの結果が良ければ:

```
この変更をコミットして PR を作成して
```

または:

```
/apply-worktree
```

結果が不要なら:

```
/delete-worktree
```

**見どころ:**
- ワークツリーから直接 commit & push → PR 作成が可能
- `/apply-worktree` でメインに変更を持ってくることもできる
- 不要なら `/delete-worktree` で跡形なく削除

#### Step 4: /best-of-n で複数モデルを比較

```
/best-of-n sonnet, gpt, composer @frontend/src/App.tsx にダークモード切替ボタンを追加して、全コンポーネントに対応させて
```

**見どころ:**
- 各モデルがそれぞれ独立したワークツリーで同じタスクを実行
- 結果をサイドバイサイドで比較できる
- 親エージェントが各結果にコメントを付けてくれる
- 最も良い結果を選んで適用できる

### ワークツリーのセットアップ設定（応用）

プロジェクトに `.cursor/worktrees.json` を置くと、ワークツリー作成時に
自動で実行するセットアップコマンドを定義できる:

```json
{
  "setup-worktree": [
    "npm ci",
    "cp $ROOT_WORKTREE_PATH/.env .env"
  ]
}
```

### 実務での活用例

- **リスクのあるリファクタ** をワークツリーで試して、安全なら取り込む
- **複数アプローチの比較:** `/best-of-n` で異なるモデルの解を比較して最善を選ぶ
- **PR 単位の分離:** 機能ごとにワークツリーを作って独立した PR を作成
- **実験的な変更:** 新しいライブラリの導入テストなど、メインを汚さずに検証

## Tips

### エディタ Agent Tab vs Agents Window の使い分け

| 状況 | 推奨 |
|------|------|
| コードを見ながら対話的に作業 | エディタの Agent Tab |
| 複数タスクを並列で同時に進めたい | Agents Window |
| 大きなタスクをバックグラウンドで放置 | Agents Window |
| UIをビジュアルで編集したい | Agents Window（Design Mode） |
| Worktree で変更を分離したい | Agents Window（`/worktree`） |

### Design Mode が有効な場面

- フロントエンド開発全般
- UI の微調整・ブラッシュアップ
- プロトタイピング
- デザインレビューの反映
